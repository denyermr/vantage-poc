## Resolution

### Substantive resolution: Option C (5-fold CV on training pool)

Phase 1c-Lean's null methodology is **5-fold CV on the n=83 training pool** with seasonal-mean computation per fold, season definition `meteorological_DJF_MAM_JJA_SON` (matching `data/splits/split_manifest.json` outer-stratification convention).

Three reasons:

1. **Symmetry with the RF baseline.** Kickoff §3.3.1 prescribed 5-fold CV on the training pool for the RF baseline. The null on the same hold-out methodology preserves comparison structure within Phase 1c-Lean's secondary-metric evaluation.

2. **No sealed-set touch.** Option B (one-time Method-1 reproduction at G3-Lean) was rejected because the G3-Lean sealed-set lock per SPEC §18.5 / DEV-1c-lean-001 is metadata-only (SHA-256 computation only). Extending the lock to "audit-pipeline-integrity verification" would set a precedent that erodes the sealed-set discipline. Sealed means sealed until conditional Block E-prime authorisation.

3. **Audit-trail preservation without sealed-set entanglement.** Both reference numbers are recorded: 0.178 (Phase 1 sealed-test, Method 1) and the locked Phase 1c-Lean training-pool 5-fold CV number (Method 3). Their non-comparability is annotated explicitly.

`baselines_locked.json` records Method 3's number to whatever precision `reproduce_baselines.py` produces at lock time. **Note on aggregation:** the halt-4 diagnostic computed pooled-RMSE (`sqrt(mean((y - y_hat)^2))` across all 83 hold-out predictions) and reported 0.1006; `reproduce_baselines.py` locks mean-of-per-fold RMSE (the standard k-fold-CV summary statistic, `mean([RMSE_fold_i for i in 1..5])`) at 0.1005.
