## Finding

Implementation drift at Session C/D authorship, not SPEC drift or
mid-stream spec change. The implementer made a "simplify first, extend
later" choice (VV-only L_physics; VH as Secondary 3 post-hoc) when
authoring the PINN composite-loss wrapper, documented the choice only in
an in-code comment, and did not author a DEV entry.

At §14 sign-off (`2331cd4`, 2026-04-19) the implementation was on disk
but untracked, so formally outside §14's scope. It entered tracked history
at F-2 close (`5a2a994`, 2026-04-20) — by which point F-2 had already run.
Two subsequent audit gates — F-1 pre-registration lock (`cea7ec9`,
2026-04-19) and F-2 λ-search kickoff (2026-04-20 pre-`5a2a994`) — had
access to the tracked or on-disk implementation but did not invoke a
cross-reference against SPEC §8. That is the audit-design gap: there
was no required implementation-audit step between §14 sign-off and F-2
kickoff, even though the on-disk implementation would be used at F-2.
F-3 entry check — specifically, a cross-reference between SPEC §8 and
`pinn_mimics.py:315` — was the audit that surfaced it, at 2026-04-20
before any Phase 1b sealed-test-set evaluation.

## Consequence for F-2

The F-2 λ search (2026-04-20) ran 64 combinations × 10 reps = 640 training
runs on the VV-only `compute_pinn_mimics_loss`. Every quantity in
`phase1b/lambda_search/results/lambda_search_f2_result.json` reflects the
VV-only formulation — including the primary-dominance evaluation
(L_data > each weighted term), the secondary-dominance evaluation
(L_physics > 10% of total loss), the tier classification
(FULL_DOMINANCE per DEV-1b-009 mean-across-reps reading), and the
selected combination (combo_idx=1, λ=(0.01, 0.01, 0.10), median val
loss 0.0353, mean physics fraction 0.435). Under the signed SPEC §8
joint VV+VH formulation, the physics-term absolute magnitude is
approximately `|VV_MSE| + |VH_MSE|` in the same dB² units, which
shifts the primary-dominance frontier and the secondary 10%-of-total
threshold. Under the corrected loss, F-2's Tier 1 FULL_DOMINANCE
classification and the selected λ are not defensible as the Phase 1b
selection.

## Resolution

**Adjudication:** The signed SPEC §8 binds (verbatim pre-registered
text rule; identical principle to DEV-1b-009). Resolution B per F-3
entry-check halt report (supervisor adjudication 2026-04-20):
amend implementation to match SPEC §8 joint VV+VH; re-run F-2 λ search
(session label F-2b); re-classify under DEV-1b-009 mean-across-reps
reading; re-select λ; F-3 replans from the new λ. Resolution A (amend
SPEC §8 post-hoc to VV-only) was considered and rejected on four
grounds: (i) inverts the DEV-1b-009 principle that was adjudicated
three days earlier under equivalent logic; (ii) SPEC §3 Δ-row names
joint VV+VH as the Phase 1b central change, so amending SPEC §8
retroactively rewrites the Phase 1b scientific contribution; (iii)
sign-chain dependencies cascade (SPEC §12 R6 risk register,
`SESSION_F_START_HERE.md` line 60, SUCCESS_CRITERIA.md §2 Secondary 3
and §6 D-3 all authored against joint); (iv) the positive outcome of
A falls out of compute-cost convenience, not out of text-linguistic
fit. Resolution C (scope narrowing) was post-hoc rationalisation and
rejected.

## Methodological observation (for Phase 1b results §6)

The cross-reference pattern invoked at F-3 entry — supervisor prompt
requiring executor grep of locked text against implementation source
before execution — is the audit that caught this. The pattern is
systematic; its activation was late. Institutionalised for Phase 1c
onward as a **required post-SPEC-sign-off implementation-audit step**,
triggered whenever an implementation file referenced by a signed SPEC
section first lands in the repo. The rule: before any experimental
session (λ search, factorial execution, diagnostic run) that uses a
tracked implementation file, the session prompt must require the
executor to grep each signed SPEC section's numerical / formulaic
claims against the implementation file's corresponding expression,
character-for-character, and record the audit outcome in the session
entry-check report.

DEV-1b-009's dual-reading mitigation was operational at F-2 but
narrowly targeted aggregation-rule interpretation, not core-loss
formulation. DEV-1b-010 generalises the discipline: verbatim text
binds for both aggregation rules *and* formulations, and the
implementation-vs-text audit step is a required gate, not an ad-hoc
executor-discretion check.
