# DEV-1b-003 — G2 Implementation Gate: published-table anchor construction and sourcing

**Date:** April 2026
**Status:** Resolved pre-sign-off (G2)
**Specification version affected:** Phase 1b SPEC.md v0.1-draft → v0.1
**Companion artefacts:** `phase1b/physics/g2_anchor_spec.md` (new, authoritative anchor specification)

---

## Trigger

G2 Implementation Gate audit during Phase 1b setup identified that SPEC §4 specifies a two-tier equivalence check ("forward-pass numerical equivalence check against a reference non-differentiable implementation") but does not resolve what reference is to be used. Two complementary arms were considered during U-1 planning:

1. **numpy_port arm** — a deterministic numpy re-implementation of the MIMICS mathematical specification, run against the differentiable PyTorch module. Catches PyTorch-specific bugs (autograd issues, broadcasting, dtype promotion, etc.).
2. **published_table arm** — anchor against σ° values published in the MIMICS literature. Catches shared-misreading bugs where both internal ports agree on a subtly incorrect equation.

At the time of Claude Code's U-1 analysis, neither candidate published source (Toure et al. 1994; McDonald et al. 1990) was staged in the repository, blocking the second arm. Claude Code presented three options: source the papers (Option A), proceed with numpy_port only and defer (Option B), or transcribe from model memory (Option C, rejected as inconsistent with the project's evidence-discipline).

## Finding

The numpy_port arm alone is a cross-framework consistency check, not a correctness check. Two ports written by the same author against the same source specification can share the same misreading — particularly for MIMICS, which is a 31-year-old IEEE paper with notation conventions (Stokes-matrix sign choices, polarimetric coordinate frames, half-space vs full-space integrals) that are easy to get wrong in either numpy or PyTorch. The published_table arm provides an external anchor that breaks this co-author failure mode.

The operational consequence is that the §11 diagnostic decision tree's first branch ("Secondary 1 fails → MIMICS implementation is wrong") becomes partly unexaminable without an external anchor. If Phase 1b returns Negative, distinguishing "MIMICS implementation is wrong" from "MIMICS is right but blanket bog breaks it" would be harder than the SPEC intends. Phase 1b is designed to be informative regardless of outcome direction; the G2 gate must match that commitment.

## Resolution

**Both source papers sourced in full via Open University Library (IEEE Xplore) on 18 April 2026 and staged in the repository:**

| Paper | DOI | Status |
|---|---|---|
| Toure, A., Thomson, K.P.B., Edwards, G., Brown, R.J., Brisco, B.G. (1994) 'Adaptation of the MIMICS backscattering model to the agricultural context — wheat and canola at L and C bands', *IEEE Transactions on Geoscience and Remote Sensing*, 32(1), 47–61. | 10.1109/36.285188 | PDF staged in repo |
| McDonald, K.C., Dobson, M.C., Ulaby, F.T. (1990) 'Using MIMICS to model L-band multiangle and multitemporal backscatter from a walnut orchard', *IEEE Transactions on Geoscience and Remote Sensing*, 28(4), 477–491. | 10.1109/TGRS.1990.572943 | PDF staged in repo |

**G2 equivalence check is redefined as a three-arm check:**

1. **numpy_port arm** (unchanged from v0.1-draft): numpy implementation agrees with PyTorch implementation within 0.5 dB across the full canonical input set. Deterministic, cross-framework consistency.

2. **published_table arm** (new, primary scientific anchor): PyTorch implementation agrees with published σ° values from Toure 1994 (primary) and McDonald 1990 (secondary) within 0.5 dB at each anchor point. Approximately 15–20 anchor values grouped into five sets (A–E), documented in `phase1b/physics/g2_anchor_spec.md`. External correctness anchor.

3. **gradient spot-check arm** (new, differentiability anchor): PyTorch autograd ∂σ°/∂parameter agrees with Toure 1994 published sensitivity coefficients (Tables V, VI) within tolerance at selected points. Unique to a differentiable PINN — proves the model is differentiable correctly, not just forward-correct.

**All three arms must pass for G2 to pass. Failure of any arm halts the experiment until the implementation is corrected** (SPEC §4 language unchanged).

## Tolerance

**0.5 dB** (inherited from SPEC §4) for the bulk-agreement anchors (Sets A, B, D). For Set C (mechanism decomposition) the same tolerance applies to each mechanism independently. For Set E (gradient check), tolerance is set relative to the sensitivity coefficient magnitude (±20% of Toure's published value, or ±0.1 dB per unit parameter, whichever is larger); this allowance reflects that Toure's published sensitivities are themselves derived from finite-difference estimates rather than closed-form derivatives.

For reference, Toure 1994 reports internal model-vs-measurement residuals of 0.62 dB (wheat CHH Etot) and 1.22 dB (canola CHH Etot). Our 0.5 dB tolerance is tighter than the paper's own model-vs-measurement bound, appropriate because we are comparing model-to-model (PyTorch vs published model) rather than model-to-measurement.

## Operating-range boundaries

Toure 1994 notes two regimes where the MIMICS model itself produces artefacts that do not match scatterometer measurements:

1. **Incidence angles < 20°:** MIMICS does not model the coherent backscatter component; measured values exceed modelled values at low angles.
2. **VV polarisation at angles ≥ 50°:** MIMICS predicts a Brewster-effect attenuation that "does not appear to be present in the data" (Toure §III).

These are properties of MIMICS, not of the measurement. Our PyTorch implementation should **reproduce these artefacts faithfully** — matching Toure's published modelled values — because doing so confirms the physics is being computed as MIMICS intends. Correspondingly, Anchor Sets A and B are bounded to 20°–50° (the interior of Toure's validated range, which also brackets Moor House's 41.5° mean incidence angle per SPEC §2). Anchor Set E sensitivities are also reported by Toure only at 20°–50°.

## Appendix — anchor refinement, April 2026

The Set A/B/C/D read-off estimates in `g2_anchor_spec.md` v0.1 were explicitly flagged as ~±0.3 dB scaffolding to be refined from the PDFs before G2 execution. Refinement was performed by rendering Toure 1994 Fig. 2 (page 49) and Fig. 7 (page 54) and McDonald 1990 Fig. 10 (page 487) at 400 dpi and reading anchor values via a mix of pixel-detection and human transcription. Set E values were extracted directly from Toure 1994 Tables IV(a) and V(a) as text via `pdftotext -layout` (no pixels). Human spot-check at A.1 (θ=20° CHH, confirmed −9.5 dB vs v0.1's −8.5 dB), B.2, B.3, B.4 (all confirmed) anchored the pixel-detection methodology.

Annotated marker-detection PNGs and read-coordinates JSON are staged at [`phase1b/refs/anchor_reads/`](refs/anchor_reads/). Final refined values are committed in `g2_anchor_spec.md` v0.2.

### Spec corrections propagated into v0.2

1. **Set B panel location.** v0.1 said "top-right" and flagged "labelled LVV … verify from PDF". Fig. 2 CVV is actually bottom-right. No numerical impact but the ambiguity is resolved.
2. **Set C marker-to-mechanism swap.** v0.1 assigned the filled-square marker to "Cover-ground" (C.3) and open-triangle to "Ground-cover" (C.4). The T94 Fig. 7 caption ("ground_cover_ground (▲); cover_ground (△); ground_cover (■); direct cover (□); direct ground (♦)") has them the other way. v0.2 uses the caption.
3. **Set C row C.3 dropped.** The open-triangle Cover-ground marker is not independently visible at CHH θ=30°. Rather than assign a conjectural value, v0.2 drops the row with this explicit record. Set C anchor count drops from 6 to 5; total anchor count drops from 23 to 22.
4. **Set C row C.5 value.** v0.1 estimated −40 dB with ±2.0 dB tolerance. Refined value is −35 dB. The v0.1 wider tolerance would still have missed the correct value by 3 dB. v0.2 uses −35 dB with unchanged ±2.0 dB tolerance.
5. **Set D figure semantics.** v0.1 described MIMICS values as "open-circle markers". M90 Fig. 10 actually draws MIMICS as lines and measurements as markers. v0.2 reads lines. All four D-rows moved by 0.3–0.8 dB from v0.1 estimates.
6. **McDonald 1990 DOI.** DEV-1b-003 cites the DOI as `10.1109/TGRS.1990.572943`. The PDF metadata reports `10.1109/TGRS.1990.572925`. Same paper. Non-blocking typo noted in `g2_anchor_spec.md` v0.2.

### Anchor count and tolerance impact

- **v0.1 advertised count:** ~15–20 (DEV-1b-003 §"Anchor-set summary"). v0.1 `g2_anchor_spec.md` specified 23 rows. v0.2 retains 22 (C.3 dropped).
- **Tolerance:** unchanged. ±0.5 dB bulk (Sets A/B/D), ±0.5 dB per-mechanism (Set C except C.5 at ±2.0 dB), ±20% or ±0.1 dB for Set E. Plus autograd-vs-FD consistency at ±5% or ±0.02 dB for Set E.
- **Three-arm structure:** unchanged.
- **Pass criteria:** unchanged (all arms must pass; all rows within tolerance).
